## PARAMETER_PLAYGROUND Data Extraction Instructions

**CRITICAL: Extract ONLY data values. DO NOT include UI hints or animation descriptions.**

### 1. Parameters Array - Extract Data Only:

**For each parameter, extract from component_specifications.interactive_elements:**

- **id**: Use descriptive ID based on question (e.g., "water_volume", "force_magnitude", "temperature")
- **label**: Extract exact label text from component_specifications (e.g., "Water Volume", "Force", "Temperature")
- **min/max**: Extract from component_specifications (e.g., min: 0, max: 10 for water volume)
  - **CRITICAL**: Must be a NUMBER, not a string (e.g., 0, not "0")
- **defaultValue**: Extract initial value from component_specifications (e.g., 0 for water added)
  - **CRITICAL**: Must be a NUMBER, not a string (e.g., 0, not "0")
- **unit**: Extract unit from component_specifications (e.g., "liters", "N", "Â°C")
- **DO NOT** include `type` field - the React component decides how to render controls

### 2. Visualization Object - Extract Data Only:

**Extract from component_specifications.visual_components:**

- **assetPrompt**: Extract detailed visualization description from component_specifications.visual_components:
  - Include component name (e.g., "beaker", "graph", "diagram")
  - Include dimensions, colors, styling details
  - Include dynamic properties (what changes based on parameters)
  - Example: "Illustrate a mystical cauldron filled with a swirling, glowing liquid representing a salt solution. Show a stream of clear liquid pouring into it, causing a change in glow intensity to visually represent concentration dilution. Include animated characters Luna and Orion providing hints."
- **DO NOT** include `type` field - the React component decides visualization type
- **DO NOT** include animation descriptions - components have built-in animations

### 3. Tasks Array - Extract Data Only:

**Extract from component_specifications.game_logic and question_flow:**

- **id**: Use descriptive ID (e.g., "task1", "dilution_task")
- **questionText**: Extract from question_flow or component_specifications (e.g., "How many liters of water must you add to achieve a 20% salt concentration?")
- **targetValues**: Extract from component_specifications.game_logic.win_condition:
  - Map parameter IDs to target values
  - **CRITICAL**: Values must be NUMBERS, not strings (e.g., { "water_volume": 5 }, not { "water_volume": "5" })
  - Example: { "water_volume": 5 } for dilution problem
- **requiredToProceed**: Set to true if task must be completed
- **DO NOT** include `type` field - the React component decides task handling

### 4. Title and NarrativeIntro - Extract Data:

**Extract from story_data:**

- **title**: Extract from story_title (e.g., "The Alchemist's Salt Solution Challenge")
- **narrativeIntro**: Extract from story_context or narrativeIntro (full narrative description)
  - **IMPORTANT**: Include full narrative text as it contains visual metaphors (e.g., "cauldron", "beaker", "tank", "alchemy", "laboratory") that the React component will use to determine visual styling and theming
  - The component will extract visual metaphors from this text to adapt colors, backgrounds, and container designs

### 5. Visual Styling Extraction:

**Extract visual metaphors and styling hints from narrativeIntro and visualization.assetPrompt:**

- **Visual Metaphors**: The narrativeIntro may contain visual metaphors (e.g., "mystical cauldron", "chemistry lab beaker", "industrial tank") that inform the component's visual design
- **Color Schemes**: Extract color hints from visualization.assetPrompt (e.g., "glowing blue liquid", "golden potion", "silver beaker")
- **Theming**: Extract theme context from narrativeIntro (e.g., "alchemy", "chemistry", "industrial", "magical")
- **DO NOT** specify exact colors or styles - the component interprets these metaphors
- **DO NOT** include `animationCues` or animation descriptions - components have built-in animations

## Key Guidelines:

1. **Extract ONLY data values**: labels, values, ranges, target values, question text
2. **DO NOT** include UI hints like `type: "slider"` or `type: "chart"`
3. **DO NOT** include `animationCues` objects or animation descriptions
4. **DO NOT** specify how components should render - components are smart and decide this
5. **Include visual metaphors** in narrativeIntro and assetPrompt - components extract styling from these
6. The React component will render high-quality UI with built-in animations based on the data you provide
