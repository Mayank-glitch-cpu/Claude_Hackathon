export interface MatchPairsBlueprint {
  templateType: "MATCH_PAIRS";
  title: string;
  narrativeIntro: string;
  pairs: Array<{
    id: string;
    leftItem: {
      id: string;
      text: string;
      imageUrl?: string;
    };
    rightItem: {
      id: string;
      text: string;
      imageUrl?: string;
    };
  }>;
  tasks: Array<{
    id: string;
    questionText: string;
    requiredToProceed: boolean;
  }>;
}

---

## MATCH_PAIRS Template Component Documentation

### Available Template Components:

1. **Card Components**:
   - **Card Pairs**: Left and right items that need to be matched
   - **Card Content**: Text or images on cards
   - **Card States**: Face-down (hidden) and face-up (revealed) states

2. **Interactive Elements**:
   - **Card Click**: Click to flip and reveal card content
   - **Card Selection**: Select two cards to attempt a match
   - **Match Connection**: Visual connection between matched pairs
   - **Reset Button**: Button to reset game state

3. **Feedback Elements**:
   - **Match Feedback**: Visual feedback when pair is correctly matched
   - **Mismatch Feedback**: Visual feedback when pair is incorrectly matched
   - **Success Message**: Appears when all pairs are matched
   - **Card Highlighting**: Visual highlighting of selected cards

## Component Customization Instructions:

### 1. Pairs Array - Customize for Question:

**Extract from component_specifications.interactive_elements:**

- **id**: Use descriptive ID based on question (e.g., "pair1", "pair2", "pair3")
- **leftItem**:
  - **id**: Use descriptive ID (e.g., "left1", "left2", "left3")
  - **text**: Extract exact text from component_specifications (e.g., "Heart", "Circulatory System", "2+2")
  - **imageUrl**: Extract image URL or description from component_specifications (if applicable)
- **rightItem**:
  - **id**: Use descriptive ID (e.g., "right1", "right2", "right3")
  - **text**: Extract exact text from component_specifications (e.g., "Pumps blood", "Blood circulation", "4")
  - **imageUrl**: Extract image URL or description from component_specifications (if applicable)

### 2. Tasks Array - Extract Data Only:

**Extract from component_specifications.game_logic and question_flow:**

- **id**: Use descriptive ID (e.g., "task1", "match_task")
- **questionText**: Extract from question_flow or component_specifications (e.g., "Match each term with its definition", "Match each equation with its solution")
- **requiredToProceed**: Set to true if task must be completed
- **DO NOT** include `type` field - the React component decides task handling
- **DO NOT** include animation descriptions - components have built-in animations

### 3. Title and NarrativeIntro - Extract Data:

**Extract from story_data:**

- **title**: Extract from story_title (e.g., "The Matching Challenge")
- **narrativeIntro**: Extract from story_context or narrativeIntro (full narrative description)
  - **IMPORTANT**: Include full narrative text as it contains visual metaphors (e.g., "matching cards", "pairing game", "memory challenge") that the React component will use to determine visual styling and theming
  - The component will extract visual metaphors from this text to adapt colors, backgrounds, and card designs

### 4. Visual Styling Extraction:

**Extract visual metaphors and styling hints from narrativeIntro:**

- **Visual Metaphors**: The narrativeIntro may contain visual metaphors (e.g., "matching cards", "pairing game", "memory challenge") that inform the component's visual design
- **Color Schemes**: Extract color hints from narrative context (e.g., "colorful cards", "memory game", "matching pairs")
- **Theming**: Extract theme context from narrativeIntro (e.g., "educational", "game", "memory", "matching")
- **DO NOT** specify exact colors or styles - the component interprets these metaphors
- **DO NOT** include `animationCues` or animation descriptions - components have built-in animations

## Key Guidelines:

1. **Extract ONLY data values**: pair text, question text
2. **DO NOT** include UI hints like `type: "card"` or `type: "match"`
3. **DO NOT** include `animationCues` objects or animation descriptions
4. **DO NOT** specify how components should render - components are smart and decide this
5. **Include visual metaphors** in narrativeIntro - components extract styling from these
6. The React component will render high-quality UI with built-in animations based on the data you provide
