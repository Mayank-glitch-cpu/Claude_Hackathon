export interface SequenceBuilderBlueprint {
  templateType: "SEQUENCE_BUILDER";
  title: string;
  narrativeIntro: string;
  steps: Array<{
    id: string;
    text: string;
    orderIndex: number; // Correct position in sequence
    description?: string;
  }>;
  distractors?: Array<{
    id: string;
    text: string;
    description?: string;
  }>;
  tasks: Array<{
    id: string;
    questionText: string;
    requiredToProceed: boolean;
  }>;
}

---

## SEQUENCE_BUILDER Template Component Documentation

### Available Template Components:

1. **Step Components**:
   - **Step Cards**: Draggable cards representing steps in a sequence
   - **Step Content**: Text or images on step cards
   - **Step Ordering**: Correct order index for validation

2. **Interactive Elements**:
   - **Drag-and-Drop**: Drag steps to reorder them
   - **Drop Zones**: Areas where steps can be placed
   - **Submit Button**: Button to validate sequence
   - **Reset Button**: Button to reset order

3. **Feedback Elements**:
   - **Correct Placement Feedback**: Visual feedback when step is in correct position
   - **Incorrect Placement Feedback**: Visual feedback when step is in wrong position
   - **Success Message**: Appears when sequence is correct
   - **Step Highlighting**: Visual highlighting of steps

## Component Customization Instructions:

### 1. Steps Array - Customize for Question:

**Extract from component_specifications.interactive_elements:**

- **id**: Use descriptive ID based on question (e.g., "step1", "step2", "step3")
- **text**: Extract exact step text from component_specifications (e.g., "Mix ingredients", "Bake at 350Â°F", "Cool for 10 minutes")
- **orderIndex**: Extract correct order from component_specifications.game_logic (0-based index)
- **description**: Extract step description from component_specifications (if applicable)

### 2. Distractors Array - Customize for Question:

**Extract from component_specifications.interactive_elements (if applicable):**

- **id**: Use descriptive ID (e.g., "distractor1", "distractor2")
- **text**: Extract distractor text from component_specifications (incorrect steps to include)
- **description**: Extract distractor description from component_specifications (if applicable)

### 3. Tasks Array - Extract Data Only:

**Extract from component_specifications.game_logic and question_flow:**

- **id**: Use descriptive ID (e.g., "task1", "sequence_task")
- **questionText**: Extract from question_flow or component_specifications (e.g., "Arrange the steps in the correct order", "Order the process steps")
- **requiredToProceed**: Set to true if task must be completed
- **DO NOT** include `type` field - the React component decides task handling
- **DO NOT** include animation descriptions - components have built-in animations

### 4. Title and NarrativeIntro - Extract Data:

**Extract from story_data:**

- **title**: Extract from story_title (e.g., "The Recipe Challenge")
- **narrativeIntro**: Extract from story_context or narrativeIntro (full narrative description)
  - **IMPORTANT**: Include full narrative text as it contains visual metaphors (e.g., "recipe cards", "workflow steps", "process flow") that the React component will use to determine visual styling and theming
  - The component will extract visual metaphors from this text to adapt colors, backgrounds, and card designs

### 5. Visual Styling Extraction:

**Extract visual metaphors and styling hints from narrativeIntro:**

- **Visual Metaphors**: The narrativeIntro may contain visual metaphors (e.g., "recipe cards", "workflow board", "process timeline") that inform the component's visual design
- **Color Schemes**: Extract color hints from narrative context (e.g., "kitchen theme", "industrial process", "scientific method")
- **Theming**: Extract theme context from narrativeIntro (e.g., "cooking", "manufacturing", "scientific", "creative")
- **DO NOT** specify exact colors or styles - the component interprets these metaphors
- **DO NOT** include `animationCues` or animation descriptions - components have built-in animations

## Key Guidelines:

1. **Extract ONLY data values**: step text, order index, question text
2. **DO NOT** include UI hints or animation descriptions
3. **DO NOT** specify how components should render - components are smart and decide this
4. **Include visual metaphors** in narrativeIntro - components extract styling from these
5. The React component will render high-quality UI with built-in animations based on the data you provide
