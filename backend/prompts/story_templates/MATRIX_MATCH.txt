### Template Context: MATRIX_MATCH

The resulting visualization will be implemented as:
- Match items across rows and columns in a matrix grid.
- Cells represent relationships between row and column items.
- Selecting row and column creates a match.

## Template Component Inventory

MATRIX_MATCH template provides the following components that can be customized for any question:

### Available Components:

1. **Matrix Components**:
   - **Matrix Grid**: Grid structure with rows and columns
   - **Row Headers**: Items in rows
   - **Column Headers**: Items in columns
   - **Cells**: Intersection points representing matches

2. **Interactive Elements**:
   - **Cell Click**: Click to select cell (row + column)
   - **Row Selection**: Select row item
   - **Column Selection**: Select column item
   - **Submit Button**: Button to validate matches
   - **Reset Button**: Button to reset selections

3. **Feedback Elements**:
   - **Correct Match Feedback**: Visual feedback when match is correct
   - **Incorrect Match Feedback**: Visual feedback when match is incorrect
   - **Success Message**: Appears when all matches are correct
   - **Cell Highlighting**: Visual highlighting of selected cells

4. **Layout Structure**:
   - **Question Container**: Top area for instructions/question text
   - **Matrix Pane**: Main area for matrix grid display
   - **Controls Pane**: Bottom area for submit/reset buttons

5. **Animation Components**:
   - **Cell Selection Animation**: Visual feedback when cell is selected
   - **Match Animation**: Success animation when match is correct
   - **Mismatch Animation**: Error animation when match is incorrect
   - **Matrix Complete Animation**: Success animation when all matches are correct

## Customization Guidance

When generating story_context, visual_metaphor, visual_elements, question_flow, animation_cues, and component_specifications:

### 1. Matrix Component Customization:
- **Matrix Size**: Extract number of rows and columns from component_specifications

- Emphasize two-dimensional relationships and correlations.
- Visual elements should represent matrix structure with rows and columns.
- Questions should involve finding matches or relationships in a matrix.
- Animation cues should show cell selection and match validation.
